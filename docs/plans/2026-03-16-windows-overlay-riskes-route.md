# Windows Overlay Teleprompter - Risks & Roadmap (Phase 5)

目标：整理当前实现计划中的风险点、依赖关系与缓解策略，形成明确的实现路线图。

风险点清单
- 渲染稳定性：跨显数量大时，文本对齐与滚动可能出现偏差。
- 鼠标穿透的兼容性：不同桌面环境、不同 Windows 版本下穿透行为可能略有差异。
- DPI/缩放问题：4K 显示器下文本清晰度与渲染效率的权衡。
- 与后端对接的可靠性：WebSocket/IPC 的断线、错误处理与重连策略。
- 多显示器场景的定位鲁棒性：显示器排列变化时的快速重新定位策略。
- 安全性：如果字幕内容来自网络，需避免潜在的注入/拦截风险。

缓解策略
- 引入自检与健康检查接口，定期校对 Overlay 的位置与文本来源的一致性。
- 对穿透实现进行全面的端到端测试，包括最常见的 Windows 版本组合。
- 将 DPI 高分辨率文本的渲染分离为独立模块，确保可替换的字体/大小配置。
- 采用稳健的断线重连机制、心跳检测和背压控制。

路线图（Phase 2-5 概览）
- Phase 2：技术栈对比完成并输出推荐方案。
- Phase 3：完成设计 tokens 及字幕样式规范。
- Phase 4：实现最小可运行框架（Electron/Qt 均可），包含文本更新接口。
- Phase 5：收敛风险、整理最终实现路线图并准备 PR/交付物。
