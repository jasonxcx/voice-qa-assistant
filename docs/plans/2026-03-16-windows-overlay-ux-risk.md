# Windows Overlay Teleprompter - Phase 5 UX/UI & Risk Roadmap

目标：整理现有提词器的 UX/UI 设计要点、核心特征，以及实现阶段的风险点与缓解策略，形成清晰的实现路线图。

一、现有提词器的核心要点（参考要点）
- 极简 UI：界面厚重感低，只提供文字信息，避免遮挡视频内容。
- 快速切换与控制：开/关、文本刷新、显示/隐藏的快捷键，支持无干扰操作。
- 字幕可读性优先：高对比、可自定义字体与字号、可切换背景透明度。
- 多源文本：支持本地文本、WebSocket/API 推送、以及本地缓存回放。
- 多显示器支持：底部居中定位、跨屏切换、并可禁用特定屏幕。
- 无障碍与可访问性：高对比度模式、文本放大、屏幕阅读器友好等。

二、用户体验设计建议
- UI 简洁的文本框：不可编辑（除非开启沙盒编辑模式），文本只读，避免带来干扰。
- 快捷键设计：Ctrl+C 复制文本、Ctrl+S 保存字幕、Ctrl+T 暂停滚动等。
- 视觉层级：字幕文本优先级高，但背景强度可调，避免遮挡视频主体。
- 自适应文本：根据屏幕分辨率和 DPI 自动调整字号与行高。
- 状态指示：显示当前源、刷新率、以及文本来源的状态指示灯。

三、实现路线图（Phase 5 风险点与缓解）
- 风险点 1：跨显示器定位在特殊屏幕布局下失效
  - 缓解：对 display-metrics-changed 事件做鲁棒处理，动态重新定位。
- 风险点 2：字幕文本更新延迟或丢帧
  - 缓解：实现本地缓存、双缓冲渲染、并发队列，确保平滑输出。
- 风险点 3：鼠标穿透在某些 Windows 版本上表现不一致
  - 缓解：对不同版本做分支适配，提供系统级配置项。
- 风险点 4：高 DPI 场景下文本模糊
  - 缓解：使用设备像素比进行字体缩放、提供 DPI 适配模式。
- 风险点 5：安全性与可维护性
  - 缓解：对 WebSocket/IPC 路径加密、错误处理与重试、清晰的 API 文档。

四、路线图输出物
- Phase 5 产出：最终实现路线图、风险矩阵、UX/UI 指南、以及对后续迭代的建议。
- 将设计要点与实现要点对齐，与 Phase 4 的实现骨架保持一致。
